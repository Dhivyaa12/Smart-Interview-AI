# 🎨 User Interface & Styling Architecture

The Smart Interview AI platform uses a hyper-modern, reactive, and fully interactive single-page application (SPA) architecture. This document outlines the component structure, styling engine, and front-end animation integrations.

---

## 1. Core Front-End Technologies
* **React 19 & Vite**: The core view layer. Leveraging React's latest concurrent rendering APIs for minimal stuttering during dense LLM streaming and complex chart rendering. Vite provides instantaneous Hot Module Replacement (HMR) during development.
* **Tailwind CSS v4**: A highly optimized utility-first CSS framework. It powers the entire design system, relying heavily on modern arbitrary variables, flexbox/grid layout constraints, and beautiful default palettes (using Indigo and Slate).
* **Framer Motion (`motion/react`)**: The physics-based animation library handling complex multi-step UI transitions (like moving from `setup` to the `interview` state) seamlessly via `<AnimatePresence>`.

---

## 2. Interactive External Libraries
* **Lucide React**: Supplies the suite of consistent, modern scalable vector graphics (SVG) icons used across the navigation (e.g., `<BarChart3 />`, `<Sparkles />`, `<Trophy />`, `<HistoryIcon />`).
* **React Markdown**: Used extensively inside the Interview and Results views to automatically render the dense Markdown responses natively generated by Gemini into HTML elements (bolding, code snippets, lists, and headers).
* **Monaco Editor (`@monaco-editor/react`)**: The engine that powers Visual Studio Code, embedded directly into the browser for the Algorithm challenges. It provides native JavaScript syntax highlighting, line numbers, and integrated editing capabilities exactly like a real IDE.
* **Recharts**: Builds the interactive SVG-based Line and Bar charts used dynamically in the Performance Analytics Dashboard.

---

## 3. View States & Component Flow
Because the application is highly cohesive, navigation is handled dynamically via state unmounting (`step` state variable) rather than independent URLs, creating an ultra-fast app-like experience.

### A. Authentication View (`step === 'auth'`)
* The landing interface. Showcases the core value proposition with a centralized layout and handles instantaneous Google OAuth invocation via Firebase popups.

### B. Interview Setup (`step === 'setup'`)
* **Design Paradigm**: Split-pane input form.
* **Left Pane**: Dynamic input fields for Target Job Role, Job Description pasting, and Native PDF File upload parsing.
* **Right Pane**: The "Recent Activity" sidebar rendering historical Firebase timestamp cards. Clicking a historical card immediately refetches and repopulates the Results View for that specific session.

### C. The Interview Queue (`step === 'interview'`)
* **Design Paradigm**: Focused, gamified floating card interfaces.
* The screen displays one question at a time. It features a pulsating microphone button denoting the Web Speech API activation state. 
* As the user interacts verbally, real-time transcription maps directly into a live-updating text area. Once submitted, the screen utilizes Framer Motion to slide the current question card out and the AI Feedback metrics in.

### D. The Coding Challenge (`step === 'coding'`)
* **Design Paradigm**: Developer standard layout.
* Mimics platforms like LeetCode. Displays the problem constraint instructions vertically on one half, and a massive, dark-themed Monaco coding window on the other. It handles the processing overlay state while the Senior AI Engineer prompt analyzes the submitted code string.

### E. Performance Analytics (`step === 'dashboard'`)
* **Design Paradigm**: Executive Summary Board.
* Automatically synthesizes raw JSON payloads stored natively in Firestore to plot chronological "Score Trends" using `<LineChart>` and detailed "Role Breakdowns" via `<BarChart>`.

### F. Comprehensive Final Results (`step === 'results'`)
* **Design Paradigm**: Report Card view.
* Features massive, bold typography mapping the combined Final Score, Technical Depth, and Communication arrays. Utilizes visually distinct cards mapping the user's raw transcript iteratively against the AI's "Improved Professional Answer". Includes a robust 1-click clipboard integration.
